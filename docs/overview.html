Overview

Drip is a base stylesheet that supports every modern writing direction, has both a light and dark scheme, and adapts to the viewport size.

Drip's default settings aim for AAA WCAG compliance. It is up to the web developer who uses this stylesheet to continue to consider accessibility as they build on top of Drip. Please do—let's create human-centered websites.

Layout

Drip adjusts the layout to the text direction and block flow direction. These must be specified on the `html` tag with the `dir` global attribute and one of Drip's flow classes. For example: 

<html lang="en" dir="ltr" class="flow-horizontal-tb">

Use semantic tags and container classes to create the basic webpage layout. The `container` class limits the width on horizontal flows or the height on vertical flows. The `container-full` class does not limit the size but does add some nice padding.

Here is an example basic layout that uses Drip: 

<!DOCTYPE html>
<html lang="en" dir="rtl" class="flow-horizontal-tb">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Example layout</title>

  <!-- Include Drip CSS -->
  <link rel="stylesheet" href="drip.css">
</head>
<body>

<header>
  <div class="container">
    <a href="#" aria-label="Home">
      <img alt="Drip logo" src="logo.svg" style="width:2rem;height:2rem;">
    </a>
    <nav>
      <ul>
        <li><a href="/" aria-current="page">Home</a></li>
        <li><a href="#">Contact</a></li>
      </ul>
    </nav>
    <div><!-- Header end position; good for locale switcher or account info --></div>
  </div>
</header>

<main class="container">
  <h1>Example layout</h1>
  <p>The main content goes here.</p>
</main>

<footer>
  <div class="container-full">
    <!-- footer content goes here -->
  </div>
</footer>

</body>
</html>

The `header`, `main`, and `footer` elements of the primary layout must be direct children of the `body` element. When these are more deeply nested, they usually have a more narrow semantic scope, so Drip styles them accordingly.

Typography

Drip sets the typography to something simple yet pleasant.

Inline elements

Drip styles links (<a href="...">) to improve accessibility, especially for the dark color scheme, and to append an icon to indicate external links.

Drip sets the size of subscripts (<sub>, as in C8H10N4O2) and superscripts (<sup>, as in 4th of July) to define a standard size for similarly sized elements, such as the external link icon on links. The size that Drip uses is close to- or the same as what browsers use as a default.

Other inline elements are not modified from the browser defaults. These that may have visual differences from regular text are listed below.

  • <abbr> — abbreviation (example: CSS)
  • <b> — bold text
  • <code> — computer code
  • <del> — deleted text
  • <dfn> — definition
  • <em> — emphasized text
  • <i> — italicized text
  • <ins> — inserted text
  • <kbd> textual user input (example: Ctrl + F5)
  • <mark> — marked text
  • <q> — quoted text
  • <ruby> — annotations (example: 漢 (Kan) 字 (ji))
  • <s> — strikethrough
  • <strong> — important text
  • <u> — non-textual annotation
  • <var> — variable

Textual block elements

Drip sets the padding and margin of textual blocks such as paragraphs and lists. This also applies to images and other media. For an example, look at the block elements in this document.
